Recon：一个可自演化、可治理的 AI 助理系统

用 stack 的方式看，这个产品不是“一个大模型 + 一个聊天框”，而是一层层叠起来的工作系统： 使用者、入口、工作空间、助理本体、运行时、能力层，再加一条始终存在的治理底座。

Overview

核心判断

真正生效的助理不是单一 profile，而是 Published Profile + Adaptive Overlays + Memory + Policy 的组合；系统价值不只在回答，还在长期学习、任务执行和可治理性。

01

产品定位

通用、可定制、可自演化、可治理的 AI 助理系统。聊天只是入口，不是产品边界。

02

工作边界

Workspace 是统一工作容器，承载 conversation、task、resource、knowledge、memory 与 profile 绑定。

03

助理本体

Profile 是稳定基线；Adaptive Overlay 是学习层；二者再与 memory、policy 组合成 effective assistant。

04

执行主线

请求先解析 actor / workspace / profile / policy，再决定直接回答、工具辅助、审批等待或转入 task。

05

能力底座

Memory 负责记住，Knowledge 负责引用和检索，Tools 负责读写和行动。

06

自演化模型

系统通过 observe → candidate → evaluate → overlay → promote / rollback 的闭环学习，而不是静默改写 profile。

07

治理约束

Identity、Safety、Evaluation、Audit、Control Plane 构成系统级约束，防止学习层越权或漂移。

08

多端交付

Web、Desktop、API、Connectors 都只是入口；核心语义必须共享同一套 request / task / approval contract。

一句话结论

Recon 的总体设计不是“一个会聊天的模型”，而是： 以 workspace 为工作边界，以 profile 为稳定基线，以 adaptive overlay 为学习层，以 memory / knowledge / tool 为能力底座，以 request / task 为执行主线，以 policy / evaluation 为治理框架 的自演化 AI 助理系统。

适合用来： 解释总体产品架构，做单页评审，向团队快速对齐核心对象、执行主线和治理边界。
