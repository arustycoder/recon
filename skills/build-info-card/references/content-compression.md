# Content Compression

Use this file when the source material is too large or too literal.

## Reduce the source to five slots

1. `eyebrow`
   One short context line.
2. `headline`
   The main claim or framing.
3. `thesis`
   The single most important judgement.
4. `modules`
   Four to nine supporting ideas.
5. `closing`
   One sentence that tells the reader how to interpret or use the card.

## Compression rules

- Write shorter than the source.
- Prefer conclusions over narration.
- Prefer labels plus one sentence over multi-paragraph explanation.
- Combine overlapping points.
- Delete caveats that do not change the main read.

## Module patterns

Each module should usually contain:

- a short label or number
- a 2-5 word title
- one sentence of interpretation

## Good signs

- The card can be understood in under 30 seconds.
- The title and thesis still make sense without the original document.
- Every module reinforces the same central judgement.

## Bad signs

- The card feels like a shrunk-down article.
- Every block is paragraph-length.
- The layout contains more categories than the audience can scan.
- The visual hierarchy does not match the conceptual hierarchy.
