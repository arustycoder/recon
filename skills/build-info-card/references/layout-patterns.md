# Layout Patterns

Use this file when the best composition is not obvious.

## Stack

Use for:

- system architecture
- top-down product explanation
- layered runtime or platform boundaries

Structure:

- eyebrow or metadata line
- large left headline
- vertical stack of 5-7 layers
- short right-side interpretation blocks
- closing thesis or foundation note

## Grid

Use for:

- parallel insights
- capability catalogs
- principles or design rules

Structure:

- hero title
- one thesis block
- 4-9 equally weighted cards
- footer with usage, scope, or references

## Compare

Use for:

- option A vs option B
- old vs new
- tradeoff framing

Structure:

- shared headline
- explicit comparison axis
- mirrored columns
- bottom recommendation

## Timeline

Use for:

- evolution
- roadmap
- implementation phases

Structure:

- timeline rail or phase markers
- short event cards
- key inflection point called out separately

## Field Guide

Use for:

- tutorial interpretation
- article summary
- practical notes

Structure:

- large title
- thesis callout
- numbered observation cards
- short footer with "use this for" and "do not confuse with"
